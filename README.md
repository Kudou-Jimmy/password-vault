# Password Vault - 密碼管理程式

幫你把所有密碼安全地存在一個地方。只需要記住一組「主密碼」，就能管理所有網站、銀行、信用卡的帳號密碼。

你的密碼**只存在你自己的電腦和瀏覽器裡**，不會上傳到任何伺服器。

> 如果你對技術細節、架構設計有興趣（或是面試想看的 HR 們），可以直接跳到最後的「[技術細節](#技術細節)」章節。

## 如何開啟這個程式

### 情況一：有人給你一個網址

如果管理員（或你自己）已經把程式部署好了，你只需要：

1. 打開你的瀏覽器（Chrome、Safari、Edge 都可以）
2. 在網址列輸入提供給你的網址，例如 `https://vault.example.com`
3. 按 Enter，程式就會開啟

**手機也一樣**——打開手機的瀏覽器，輸入同一個網址就能用。

### 情況二：自己在電腦上執行

如果你拿到的是程式的原始檔案（一個資料夾），需要先在電腦上啟動它。

#### 第一次使用前：安裝 Node.js

Node.js 是讓這個程式能在你電腦上跑起來的工具，只需要安裝一次。

1. 用瀏覽器打開 https://nodejs.org
2. 點畫面上綠色的「Download」按鈕下載
3. 下載完雙擊安裝檔
4. 一路點「下一步」（Next）直到安裝完成
5. 安裝完後**重新啟動電腦**（建議，確保生效）

#### 第一次使用前：下載程式檔案

**方法 A：直接下載壓縮檔**
1. 到本專案的 GitHub 頁面
2. 點綠色的「Code」按鈕 → 點「Download ZIP」
3. 下載後解壓縮，把資料夾放到你方便的位置（例如桌面）

**方法 B：用 Git 下載（如果你有裝 Git）**
```
git clone https://github.com/Kudou-Jimmy/password-vault.git
```

#### 第一次使用前：安裝程式所需的套件

1. 打開終端機（命令列）：

   **Windows 方法**：
   - 打開程式資料夾
   - 在資料夾空白處，按住 `Shift` 鍵再按`右鍵`
   - 點選「在此處開啟命令視窗」或「在終端中開啟」
   - 會跳出一個黑色（或藍色）的文字視窗

   **Mac 方法**：
   - 打開「終端機」應用程式（在「應用程式」→「工具程式」裡，或用 Spotlight 搜尋「終端機」）
   - 輸入 `cd `（注意 cd 後面有一個空格）
   - 然後把程式資料夾**從 Finder 拖進**終端機視窗裡
   - 按 Enter

2. 輸入以下指令，然後按 Enter：
   ```
   npm install
   ```
3. 等它跑完（畫面會跳出很多文字，這是正常的，大約需要一兩分鐘）
4. 看到類似 `added xxx packages` 的訊息就代表完成了

> 以上這些步驟**只需要做一次**，以後啟動程式不用再做。

#### 啟動程式

每次要使用時，做以下步驟：

1. 打開終端機，進入程式資料夾（方法同上）
2. 輸入以下指令，然後按 Enter：
   ```
   npm run dev
   ```
3. 等幾秒鐘，畫面會顯示類似這樣的內容：
   ```
   VITE v8.0.8  ready in 374 ms

   ➜  Local:   http://localhost:5173/
   ➜  Network: http://192.168.1.100:5173/
   ```
4. 打開你的瀏覽器（Chrome、Safari、Edge 都可以）
5. 在網址列輸入 `http://localhost:5173/` 然後按 Enter
6. 程式就開啟了！

> **什麼是 `localhost:5173`？**
> `localhost` 是指「這台電腦自己」的意思，`5173` 是程式使用的通訊埠號碼。這不是一個真正的網站，只有在你的電腦上才能打開。

#### 關閉程式

用完之後：
1. 回到終端機視窗
2. 按 `Ctrl + C`（Windows 和 Mac 都一樣），程式就會停止
3. 關閉終端機視窗即可

> **注意：終端機關掉或電腦關機後，`localhost:5173` 就會連不上。**
> 這是正常的，下次要用再重新執行 `npm run dev` 就好。
> 你存的密碼不會消失，它們安全地保存在瀏覽器裡。

#### 手機或平板要怎麼連？

1. 確保手機和電腦連在**同一個 Wi-Fi**
2. 程式啟動時，看終端機畫面上 `Network:` 後面的網址，例如：
   ```
   ➜  Network: http://192.168.1.100:5173/
   ```
3. 在手機瀏覽器輸入那個網址就能用
4. 每台電腦的 Network 網址不同，**請以你畫面上顯示的為準**

> **注意：手機和電腦必須在同一個 Wi-Fi 下才能連線。**
> 如果出門在外想用手機存取，需要額外設定部署（請參考技術細節的「部署方式」章節）。

## 使用說明

### 第一步：建立你的密碼金庫

第一次打開程式，你會看到這個畫面：

```
🔐 建立密碼金庫
設定一組主密碼來保護你的資料

[________________]  ← 輸入你想設定的主密碼
[________________]  ← 再輸入一次確認

[   建立金庫   ]
```

- **主密碼**就像一把鑰匙，之後每次打開程式都要輸入它
- 至少要 8 個字，建議用一句你容易記住但別人猜不到的話
- 例如：`我家小黑2019年來的!`

> **請務必牢記你的主密碼！忘記的話，裡面的資料就再也打不開了。**
> 建議把主密碼寫在紙上，放在家裡安全的地方（例如抽屜裡、保險箱裡）。

---

### 第二步：存入你的密碼

進入主畫面後，點右上角的「**+ 新增**」按鈕，選擇你要存什麼：

#### 🔑 一般網站帳號（Facebook、Google、蝦皮...）

選「**登入**」，填入：
- **標題** → 例如「Facebook」
- **網站** → 例如 `facebook.com`
- **帳號** → 你的帳號
- **密碼** → 你的密碼
- 填完按「**儲存**」

#### 🏦 網路銀行

選「**網路銀行**」，填入：
- **標題** → 例如「中國信託網銀」
- **銀行名稱** → 中國信託
- **身分證字號** → 你的身分證字號
- **使用者代號** → 銀行給你的代號
- **使用者密碼** → 你的網銀密碼

#### 🏧 金融卡（提款卡）

選「**金融卡**」，填入：
- **標題** → 例如「郵局金融卡」
- **銀行名稱** → 郵局
- **提款密碼** → ATM 提款的密碼
- **無卡提款密碼** → 如果有設定的話
- **磁條密碼** → 出國領錢用的密碼

#### 💳 信用卡

選「**信用卡**」，填入：
- **標題** → 例如「玉山 U Bear 卡」
- **卡號** → 信用卡卡號
- **持卡人** → 卡片上的英文名字
- **到期日** → 例如 `12/27`
- **CVV** → 卡片背面 3 碼數字
- **預借現金密碼** → 如果有設定的話

#### 👤 身份資料、📝 加密筆記

也可以存個人資料（姓名、地址、電話）或任何需要加密保存的文字筆記。

---

### 第三步：查看和複製密碼

需要用到密碼的時候：

1. 在左邊的列表中，**點一下**你要找的項目
2. 右邊會顯示詳細資料
3. 密碼預設會顯示成 `••••••••`（小圓點），這是為了保護你
4. 點「**顯示**」→ 就能看到密碼
5. 點「**複製**」→ 密碼會複製起來，到其他地方按「貼上」（Ctrl+V 或 Command+V）就好
6. **不用擔心**：複製的密碼會在 30 秒後自動從剪貼簿清掉

### 搜尋密碼

密碼太多找不到？在列表上方的搜尋框打幾個字就好：

```
🔍 [中國信託___]
```

打「中國」就會找出所有標題或內容有「中國」的項目。

### 用資料夾整理

密碼越來越多的時候，可以用資料夾分類：

1. 在左邊欄位的「資料夾」旁邊，點「**+**」
2. 輸入名稱，例如「銀行」「社群」「購物」
3. 新增或編輯密碼時，選擇要放在哪個資料夾

也可以用「標籤」做更靈活的分類。編輯項目時，在標籤欄位輸入文字即可（多個標籤用逗號隔開，例如 `工作, 重要`）。

### 幫你產生安全的密碼

很多人用 `123456` 或 `password` 當密碼，這些幾秒鐘就能被破解。程式可以幫你產生很強的密碼：

1. 新增或編輯密碼時，點「**🎲 密碼產生器**」
2. 會自動產生一組像這樣的密碼：`k8#Qm!pL2xR$nW9v`
3. 可以調整密碼長度（拉桿左右拉）
4. 滿意的話，點「**使用**」就會自動填入密碼欄位

也可以選「**助記詞組**」模式，產生像這樣比較好記的密碼：
```
brave-ocean-tiger-flame
```

不用擔心記不住——因為你已經把它存在金庫裡了，需要的時候來查就好。

### 檢查你的密碼安不安全

點列表上方的「**📊**」圖示，程式會幫你做一份「健康報告」：

- 哪些密碼太弱，容易被破解
- 哪些密碼重複使用（很危險！一個被盜，全部都會被盜）
- 哪些密碼太久沒換（超過 90 天）

### 備份你的密碼（很重要！）

密碼存在瀏覽器裡，如果清除瀏覽器資料、換電腦、或重灌系統，密碼就會不見。**所以一定要定期備份！**

**備份方法：**
1. 點列表上方的「**📤**」圖示
2. 選「**🔒 加密匯出**」
3. 會下載一個檔案（檔名像 `vault-1713012345678.json`）
4. 把這個檔案存到安全的地方，例如：
   - USB 隨身碟
   - iCloud 雲碟
   - Google 雲端硬碟
   - 寄 email 給自己

**還原備份（在新電腦或新瀏覽器）：**
1. 打開程式，在登入畫面點下方的「**匯入既有金庫**」
2. 選擇你備份的那個檔案
3. 輸入你的主密碼，就能恢復所有密碼

**明碼匯出（進階）：**

如果你想把密碼匯出成一般的表格檔案（可以用 Excel 開）：
1. 點「**⚠️ 明碼匯出**」
2. 選 CSV 或 JSON 格式
3. **注意：這個檔案裡的密碼是可以直接看到的，千萬不要傳給別人或放在不安全的地方！**

## 常見問題

**Q：忘記主密碼怎麼辦？**
A：很遺憾，無法找回。這是為了安全設計的——如果有人拿到你的電腦，他們也無法在不知道主密碼的情況下看到你的密碼。建議把主密碼寫在紙上，放在家裡安全的地方。

**Q：換電腦或換瀏覽器，密碼還在嗎？**
A：不會自動帶過去。請先用「📤 加密匯出」備份，再到新電腦「匯入既有金庫」。

**Q：手機可以用嗎？**
A：可以。用手機的瀏覽器打開同一個網址就能用，畫面會自動適應手機螢幕大小。

**Q：安全嗎？**
A：你的密碼用軍事等級的加密技術保護（AES-256-GCM），只有知道主密碼的人才能解開。而且所有資料都存在你自己的瀏覽器裡，不會傳到任何外部伺服器。

**Q：需要付費嗎？**
A：完全免費，開放原始碼。

**Q：清除瀏覽器資料或記錄，密碼會不見嗎？**
A：會！密碼存在瀏覽器的本地儲存空間裡，清除瀏覽資料時可能會一起被清掉。所以請務必定期備份。

**Q：多人可以共用同一個金庫嗎？**
A：可以。把加密匯出的檔案分享給對方，對方匯入後用相同的主密碼就能開啟。但請注意，各自修改後資料不會自動同步。

### 加密設計

#### 金鑰衍生

```
主密碼 (UTF-8)
    │
    ▼
  PBKDF2
    ├── 雜湊函式：SHA-256
    ├── 迭代次數：600,000 次
    ├── Salt：32 bytes（每次加密隨機產生）
    │
    ▼
  AES-256 金鑰 (256 bits)
```

- **PBKDF2（Password-Based Key Derivation Function 2）**：將人類可讀的密碼轉換為加密用的金鑰
- **600,000 次迭代**：符合 OWASP 2023 建議的最低標準，刻意減慢衍生速度以抵禦暴力破解
- **隨機 Salt**：每次加密使用獨立的 32 bytes 隨機 Salt，即使相同密碼也會產生不同金鑰

#### 資料加密

```
明文 JSON
    │
    ▼
  AES-256-GCM
    ├── 金鑰：由 PBKDF2 衍生的 256-bit 金鑰
    ├── IV：12 bytes（每次加密隨機產生）
    ├── 模式：GCM（Galois/Counter Mode）
    │
    ▼
  密文 + Authentication Tag
```

- **AES-256-GCM**：美國國家標準與技術研究院（NIST）認證的對稱加密演算法
- **GCM 模式**：同時提供加密（confidentiality）與完整性驗證（authenticity），偵測任何篡改
- **隨機 IV**：每次加密使用獨立的 12 bytes IV，避免 nonce 重複攻擊
- **使用瀏覽器原生 Web Crypto API**：硬體加速、經過安全審計，不依賴第三方 JavaScript 加密庫

#### 儲存格式

金庫以單一 JSON 物件存於 localStorage：

```json
{
  "version": 1,
  "salt": "Base64 編碼的 32-byte Salt",
  "iv": "Base64 編碼的 12-byte IV",
  "data": "Base64 編碼的 AES-256-GCM 密文"
}
```

- `version` 欄位確保未來更換加密方式或資料結構時，能向後相容地進行資料遷移
- 整個金庫（所有條目、資料夾、標籤、歷史紀錄）作為一個整體加密，不單獨加密個別欄位

### 資料模型

#### 條目（Entry）

```typescript
interface VaultEntry {
  id: string;          // UUID v4，全域唯一識別碼
  type: EntryType;     // 'login' | 'bank' | 'debit-card' | 'card' | 'identity' | 'note'
  title: string;       // 使用者定義的標題
  fields: VaultField[]; // 動態欄位陣列（見下方）
  tags: string[];      // 標籤陣列
  folder: string;      // 所屬資料夾
  favorite: boolean;   // 是否為最愛
  history: HistoryRecord[]; // 修改歷史
  createdAt: number;   // 建立時間戳（Unix ms）
  updatedAt: number;   // 最後修改時間戳（Unix ms）
  deletedAt: number | null; // 軟刪除時間戳，null 表示未刪除
}

interface VaultField {
  key: string;         // 欄位名稱（如「密碼」「帳號」）
  value: string;       // 欄位值
  sensitive: boolean;  // 是否為敏感欄位（UI 預設遮蔽）
}
```

**動態欄位設計**：欄位以 `VaultField[]` 陣列儲存，而非固定的 `username` / `password` 屬性。這使得：
- 不同條目類型可有不同欄位結構
- 使用者可自行新增/刪除欄位
- 新增條目類型不需要修改資料結構（schema）

#### 條目類型與預設欄位

| 類型 | 代碼 | 預設欄位 |
|------|------|----------|
| 登入 | `login` | 網站、帳號、密碼、備註 |
| 網路銀行 | `bank` | 銀行名稱、身分證字號、使用者代號、使用者密碼、備註 |
| 金融卡 | `debit-card` | 銀行名稱、卡號、提款密碼、無卡提款密碼、磁條密碼、備註 |
| 信用卡 | `card` | 卡號、持卡人、到期日、CVV、預借現金密碼、備註 |
| 身份 | `identity` | 姓名、電子郵件、電話、地址、備註 |
| 筆記 | `note` | 內容 |

### 安全機制

| 機制 | 實作方式 |
|------|----------|
| 加密演算法 | AES-256-GCM（NIST 認證，提供加密 + 完整性驗證） |
| 金鑰衍生 | PBKDF2-SHA256，600,000 次迭代（OWASP 2023 建議） |
| 隨機數產生 | `crypto.getRandomValues()`（CSPRNG，密碼學安全亂數） |
| 零知識架構 | 伺服器僅提供靜態檔案，不接收、不處理、不儲存任何使用者資料 |
| 自動鎖定 | 閒置 5 分鐘後清除記憶體中的主密碼和解密資料 |
| 剪貼簿保護 | 複製密碼後 30 秒自動清除剪貼簿內容 |
| 軟刪除 | 刪除操作為軟刪除（設定 `deletedAt` 時間戳），可從回收桶恢復 |
| 密碼強度評估 | 使用 zxcvbn 演算法，基於常見密碼字典、鍵盤模式、l33t 替換等多維度評估 |

### 事件系統

核心操作透過發布/訂閱模式發出事件，功能模組可訂閱事件而不耦合核心邏輯：

```typescript
type EventType =
  | 'entry.created'          // 條目新增
  | 'entry.updated'          // 條目修改（自動觸發歷史紀錄）
  | 'entry.deleted'          // 條目軟刪除
  | 'entry.restored'         // 條目從回收桶恢復
  | 'entry.permanentlyDeleted' // 條目永久刪除
  | 'vault.locked'           // 金庫鎖定
  | 'vault.unlocked'         // 金庫解鎖
  | 'vault.exported'         // 金庫匯出
  | 'vault.imported';        // 金庫匯入
```

**擴充範例**：未來新增「操作日誌」功能時，只需訂閱相關事件，不需修改任何現有程式碼。

### 密碼產生器

提供兩種模式：

| 模式 | 實作 | 熵值範例 |
|------|------|----------|
| 隨機字元 | 從字元集（大小寫字母 + 數字 + 符號，共 94 字元）中以 CSPRNG 隨機取字元 | 20 字元 ≈ 131 bits |
| 助記詞組 | 從 52 個英文單字表中以 CSPRNG 隨機取詞，用分隔符連接 | 4 個詞 ≈ 22.8 bits |

兩種模式皆使用 `crypto.getRandomValues()` 產生密碼學安全的亂數。

### 擴充性設計

本專案在架構上保留了以下擴充點：

1. **動態欄位**：新增條目類型只需在 `DEFAULT_FIELDS` 增加定義，不需修改資料模型
2. **事件系統**：新功能可透過訂閱事件被動觸發，不修改核心邏輯
3. **金庫版本化**：`version` 欄位支援未來資料遷移
4. **模組化結構**：功能模組位於 `features/` 目錄，與核心和 UI 解耦
5. **Context 分離**：金庫狀態與主題狀態各自獨立，互不影響
